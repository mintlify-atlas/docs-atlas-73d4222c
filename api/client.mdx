The `@discordkit/client` package provides a complete, type-safe interface to Discord's REST API. Each endpoint is exported with multiple variants (base function, schema, safe function, procedure, and query) following the [export patterns](/api/overview#export-patterns) described in the overview.

## Installation

```bash
npm install @discordkit/client valibot
```

<Note>
  DiscordKit uses [Valibot](https://valibot.dev/) for schema validation and requires it as a peer dependency.
</Note>

## Initialization

Before making requests, initialize the Discord session with your bot token:

```typescript
import { discord } from '@discordkit/client';

// Initialize with bot token
discord.setToken('Bot YOUR_BOT_TOKEN_HERE');

// Or with Bearer token for OAuth2
discord.setToken('Bearer YOUR_OAUTH_TOKEN');
```

## Core Export

### discord

The main `DiscordSession` instance that manages authentication and rate limiting.

<ParamField path="discord" type="DiscordSession">
  Global Discord API session manager. Re-exported from `@discordkit/core`.
</ParamField>

**Methods:**

<ResponseField name="setToken" type="(token: string) => DiscordSession">
  Set the authentication token. Must be prefixed with `Bot ` or `Bearer `.
</ResponseField>

<ResponseField name="clearSession" type="() => DiscordSession">
  Clear authentication and reset rate limit tracking.
</ResponseField>

<ResponseField name="getSession" type="() => string">
  Get the current authentication token. Throws if not set.
</ResponseField>

<ResponseField name="queueRequest" type="(resource: URL, method: string, body?: string) => Promise<Response>">
  Queue an HTTP request with automatic rate limiting.
</ResponseField>

<ResponseField name="getQueueSize" type="() => number">
  Get the current number of queued requests.
</ResponseField>

**Properties:**

<ResponseField name="ready" type="boolean">
  Returns `true` if an auth token is set.
</ResponseField>

<ResponseField name="endpoint" type="string" default="https://discord.com/api/v10/">
  Base URL for Discord API requests.
</ResponseField>

<ResponseField name="maxRetries" type="number" default={5}>
  Maximum retry attempts for rate-limited requests.
</ResponseField>

## Usage Examples

### Basic Request

```typescript
import { discord, getGuild } from '@discordkit/client';

discord.setToken('Bot YOUR_BOT_TOKEN');

const guild = await getGuild({
  id: '123456789',
  params: { withCounts: true }
});

console.log(guild.name, guild.approximateMemberCount);
```

### With Runtime Validation

```typescript
import { discord, getGuildSafe } from '@discordkit/client';

discord.setToken('Bot YOUR_BOT_TOKEN');

// Validates input and output at runtime
const guild = await getGuildSafe({
  id: '123456789',
  params: { withCounts: true }
});
```

### With tRPC

```typescript
import { initTRPC } from '@trpc/server';
import { getGuildProcedure, createGuildProcedure, discord } from '@discordkit/client';

const t = initTRPC.create();

// Initialize Discord session in context
export const createContext = () => {
  discord.setToken('Bot YOUR_BOT_TOKEN');
  return {};
};

const appRouter = t.router({
  getGuild: getGuildProcedure(t.procedure),
  createGuild: createGuildProcedure(t.procedure),
});

export type AppRouter = typeof appRouter;
```

### With React Query

```typescript
import { useQuery } from '@tanstack/react-query';
import { getGuildQuery, discord } from '@discordkit/client';

// Initialize once at app startup
discord.setToken('Bot YOUR_BOT_TOKEN');

function GuildDisplay({ guildId }: { guildId: string }) {
  const { data: guild, isLoading } = useQuery({
    queryKey: ['guild', guildId],
    queryFn: getGuildQuery({ id: guildId, params: { withCounts: true } })
  });

  if (isLoading) return <div>Loading...</div>;
  return <div>{guild?.name}</div>;
}
```

## Rate Limiting

The `discord` session automatically handles rate limiting:

- **Global rate limit**: 50 requests per second
- **Per-route rate limits**: Automatically tracked via Discord headers
- **429 responses**: Automatically retried with exponential backoff
- **Request queuing**: Requests are queued and processed when rate limits allow

You can monitor the queue size:

```typescript
import { discord } from '@discordkit/client';

const queueSize = discord.getQueueSize();
console.log(`Queued requests: ${queueSize}`);
```

## Error Handling

```typescript
import { discord, getGuild } from '@discordkit/client';

discord.setToken('Bot YOUR_BOT_TOKEN');

try {
  const guild = await getGuild({ id: '123456789' });
} catch (error) {
  if (error instanceof Error) {
    console.error('Failed to fetch guild:', error.message);
  }
}
```
