DiscordKit provides helper functions to generate properly formatted Discord CDN URLs for various asset types. These helpers handle URL construction, format selection, and size parameters automatically.

## Overview

All image helpers generate URLs pointing to Discord's CDN (`https://cdn.discordapp.com`). They support multiple image formats and size options, making it easy to retrieve optimized assets for your application.

### Common Parameters

Most image helpers accept the following optional parameters:

- **format**: The image format (`png`, `jpg`, `webp`, or `gif` where supported)
  - Defaults to `png` if not specified
  - Not all assets support all formats (see individual functions)
- **params.size**: The desired image size in pixels
  - Must be one of: `16`, `32`, `64`, `128`, `256`, `512`, `1024`, `2048`, `4096`
  - Discord will return the closest available size

## User Assets

### userAvatar

Generates a CDN URL for a user's avatar.

```typescript
import { userAvatar } from '@discordkit/client';

const url = userAvatar({
  user: '123456789012345678',
  avatar: 'a_1234567890abcdef',
  format: 'png',
  params: { size: 256 }
});
// https://cdn.discordapp.com/avatars/123456789012345678/a_1234567890abcdef.png?size=256
```

**Parameters:**
- `user` (string): The user's snowflake ID
- `avatar` (string): The avatar hash from the user object
- `format` (optional): `png`, `jpg`, `webp`, or `gif`
- `params.size` (optional): Image size in pixels

Generates a CDN URL for Discord's default user avatars.


**Parameters:**
- `index` (string): The default avatar index (typically calculated from the user's discriminator)

**Note:** Default avatars are always PNG format and don't support size parameters.

## Best Practices

### Choose the Right Format

- Use **WebP** for the best compression and quality balance (when supported)
- Use **GIF** only when you need animated images
- Use **PNG** for maximum compatibility
- Use **JPG** when you need broad support for static images

### Optimize Image Sizes

Request the smallest size that meets your needs to reduce bandwidth and improve loading times:

```typescript
// For small thumbnails
userAvatar({ user, avatar, params: { size: 64 } })

// For profile displays
userAvatar({ user, avatar, params: { size: 256 } })

// For large displays
userAvatar({ user, avatar, params: { size: 512 } })
```

### Handle Missing Assets

Always check if the asset hash exists before generating URLs:

```typescript
const avatarUrl = user.avatar
  ? userAvatar({ user: user.id, avatar: user.avatar, format: 'webp' })
  : defaultUserAvatar({ index: String(Number(user.id) % 5) });
```

### Animated Images

Animated avatars have hashes starting with `a_`. Use GIF format to display animations:

```typescript
const format = avatar.startsWith('a_') ? 'gif' : 'webp';
const url = userAvatar({ user, avatar, format });
```
