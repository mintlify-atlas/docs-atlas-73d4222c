Webhooks are a way to send messages to Discord channels without a bot. All webhook endpoints support audit log reasons via the `X-Audit-Log-Reason` header where noted.

## Available Endpoints

### Create Webhook

Create a new webhook and return a webhook object on success.

**POST** `/channels/:channel/webhooks`

```typescript
import { createWebhook, createWebhookSchema } from "@discordkit/client";

const webhook = await createWebhook({
  channel: "123456789",
  body: {
    name: "My Webhook",
    avatar: "https://example.com/avatar.png"
  }
});
```

**Permissions:** Requires `MANAGE_WEBHOOKS` permission.

**Supports Audit Log Reason:** Yes

**Exports:**
- `createWebhook` - Handler function
- `createWebhookSchema` - Valibot schema for validation
- `createWebhookSafe` - Validated handler
- `createWebhookProcedure` - tRPC procedure (mutation)

### Delete Webhook Message

**DELETE** `/webhooks/:webhook/:token/messages/:message`

```typescript

  webhook: "123456789",
```

### Edit Webhook Message

Edit a previously-sent webhook message from the same token.

**PATCH** `/webhooks/:webhook/:token/messages/:message`

```typescript
import { editWebhookMessage } from "@discordkit/client";

const message = await editWebhookMessage({
  webhook: "123456789",
  token: "webhook_token",
  message: "987654321",
  params: {
    threadId: "111111111", // Optional
    withComponents: true // Optional
  },
  body: {
    content: "Updated content",
    embeds: [{
      title: "Updated Embed",
      description: "New description",
      type: "rich"
    }]
  }
});
```

**Note:** All parameters in the body are optional and nullable. The `attachments` array must contain all attachments that should be present after edit, including retained and new attachments.

**Exports:**
- `editWebhookMessage` - Handler function
- `editWebhookMessageSchema` - Valibot schema for validation
- `editWebhookMessageSafe` - Validated handler
- `editWebhookMessageProcedure` - tRPC procedure (mutation)

### Execute GitHub-Compatible Webhook

Execute a webhook configured for GitHub compatibility.

**POST** `/webhooks/:webhook/:token/github`

```typescript
import { executeGitHubCompatibleWebhook } from "@discordkit/client";

await executeGitHubCompatibleWebhook({
  webhook: "123456789",
  token: "webhook_token",
  params: {
    threadId: "111111111", // Optional
    wait: true // Optional: defaults to true
  }
});
```

**Supported Events:** `commit_comment`, `create`, `delete`, `fork`, `issue_comment`, `issues`, `member`, `public`, `pull_request`, `pull_request_review`, `pull_request_review_comment`, `push`, `release`, `watch`, `check_run`, `check_suite`, `discussion`, `discussion_comment`.

**Exports:**
- `executeGitHubCompatibleWebhook` - Handler function
- `executeGitHubCompatibleWebhookSchema` - Valibot schema for validation
- `executeGitHubCompatibleWebhookSafe` - Validated handler
- `executeGitHubCompatibleWebhookProcedure` - tRPC procedure (mutation)

### Execute Webhook

Execute a webhook to send a message.

**POST** `/webhooks/:webhook/:token`

```typescript
import { executeWebhook } from "@discordkit/client";

await executeWebhook({
  webhook: "123456789",
  token: "webhook_token",
  params: {
    wait: false, // Optional: defaults to false
    threadId: "111111111", // Optional
    withComponents: true // Optional: defaults to false
  },
  body: {
    content: "Hello, World!",
    username: "Custom Username",
    avatarUrl: "https://example.com/avatar.png",
    tts: false,
    embeds: [{
      title: "Example Embed",
      description: "This is an embed",
      type: "rich"
    }],
    components: [],
    threadName: "New Thread", // For forum channels
    appliedTags: ["tag_id"], // For forum/media channels
    poll: {
      question: { text: "What's your favorite color?" },
      answers: [
        { answer_id: 1, poll_media: { text: "Red" } },
        { answer_id: 2, poll_media: { text: "Blue" } }
      ],
      duration: 24
    }
  }
});
```

**Note:** You must provide at least one of `content`, `embeds`, `components`, or `file`.

**Exports:**
- `executeWebhook` - Handler function
- `executeWebhookSchema` - Valibot schema for validation
- `executeWebhookSafe` - Validated handler
- `executeWebhookProcedure` - tRPC procedure (mutation)

## Procedure Exports

All webhook procedures are also exported as a namespace:

```typescript

// Access any procedure:
webhookProcedures.createWebhookProcedure
webhookProcedures.deleteWebhookProcedure
// ... etc
```
