## Overview

DiscordKit uses [Valibot](https://valibot.dev) for runtime schema validation, providing type-safe validation for both input parameters and API responses. This ensures your application handles data correctly and catches errors early.

## Why Validation?

Runtime validation provides:

- **Type Safety**: Catch type errors at runtime, not just compile time
- **Data Integrity**: Ensure API responses match expected schemas
- **Better Errors**: Get detailed validation errors instead of generic failures
- **User Input Safety**: Validate user-provided data before sending to Discord

## Validation Variants

Every DiscordKit fetcher comes in three variants:

<Tabs>
  <Tab title="Standard (Unvalidated)">
    The default variant with no runtime validation:

    ```typescript
    import { getGlobalApplicationCommands } from '@discordkit/client';

    // Fast, but no validation
    const commands = await getGlobalApplicationCommands({
      application: '123456789'
    });
    ```

    <Note>
      Use this for trusted data or when performance is critical.
    </Note>
  </Tab>

  <Tab title="Safe (Validated)">
    The `*Safe` variant validates both input and output:

    ```typescript
    import { getGlobalApplicationCommandsSafe } from '@discordkit/client';

    // Validates input schema and output schema
    const commands = await getGlobalApplicationCommandsSafe({
      application: '123456789'
    });
    // Throws detailed error if validation fails
    ```

    <Note>
      Use this when working with user input or untrusted data.
    </Note>
  </Tab>

  <Tab title="Custom Validation">
    Create custom validated variants with `toValidated`:

    ```typescript
    import { toValidated, getCurrentApplication } from '@discordkit/client';
    import { applicationSchema } from '@discordkit/client/application';

    const customValidated = toValidated(
      getCurrentApplication,
      null, // No input schema
      applicationSchema // Output schema
    );
    ```
  </Tab>
</Tabs>

### How It Works

The `toValidated` function uses a Proxy to intercept calls:

1. **Validate Input**: Check input against the input schema
2. **Execute Request**: Call the original fetcher
3. **Validate Output**: Check response against the output schema
4. **Return or Throw**: Return data if valid, throw detailed error if not

```typescript
export const toValidated = <F extends Fetcher>(
  fn: F,
  input?: GenericSchema | null,
  output?: GenericSchema
): F =>
  new Proxy<F>(fn, {
    async apply(target, _, [config]): Promise<ReturnType<F>> {
      // Validate input
      if (input && isOfKind(`schema`, input)) {
        const { issues } = await safeParseAsync(input, config);
        if (issues) {
          throw new Error(
            `Failed to parse input schema: ${input.reference.name}\n\n${summarize(issues)}`
          );
        }
      }

      const result = await target(config);

      // Validate output
      if (output && isOfKind(`schema`, output)) {
        const { issues } = await safeParseAsync(output, result);
        if (issues) {
          throw new Error(
            `Failed to parse output schema: ${output.reference.name}\n\n${summarize(issues)}`
          );
        }
      }

      return result as ReturnType<F>;
    }
  });
```

## Built-in Validators

DiscordKit provides custom Valibot validators for Discord-specific types:

### Snowflake

Validates Discord snowflake IDs:

```typescript
import * as v from 'valibot';
import { snowflake } from '@discordkit/core';

const schema = v.object({
  userId: snowflake,
  guildId: snowflake,
  channelId: snowflake
});

// Valid
v.parse(schema, {
  userId: '123456789012345678',
  guildId: '987654321098765432',
  channelId: '111111111111111111'
});

// Invalid - throws error
v.parse(schema, {
  userId: 'not-a-snowflake' // Error: Invalid Snowflake
});
```

**Implementation:**

```typescript
export const snowflake: v.GenericSchema<string> = v.pipe(
  v.custom<string, `Invalid Snowflake`>(
    (val) =>
      isNonNullable(val) &&
      (typeof val === `bigint` ||
        typeof val === `number` ||
        isNumericString(val)) &&
      snowflakeToDate(val).getTime() >= DISCORD_EPOCH,
    `Invalid Snowflake`
  ),
  v.title(`snowflake`)
);
```

### Bounded String

Validates strings within length constraints:

```typescript
import { boundedString } from '@discordkit/core';
import * as v from 'valibot';

const schema = v.object({
  username: boundedString({ min: 2, max: 32 }),
  bio: boundedString({ max: 190 }),
  tag: boundedString(4) // Exact length
});

// Valid
v.parse(schema, {
  username: 'CoolUser',
  bio: 'I love coding',
  tag: '0001'
});

// Invalid
v.parse(schema, {
  username: 'a', // Too short
  bio: 'x'.repeat(200), // Too long
  tag: '001' // Wrong length
});
```

**Implementation:**

```typescript
export const boundedString = (
  req: number | { min?: number; max?: number } = {}
): v.GenericSchema<string> =>
  v.message<v.GenericSchema<string>>(
    typeof req === `number`
      ? v.pipe(v.string(), v.length(req))
      : typeof req.max === `number`
        ? v.pipe(v.string(), v.minLength(req.min ?? 1), v.maxLength(req.max))
        : v.pipe(v.string(), v.minLength(req.min ?? 1)),
    (issue) =>
      `Expected a string with a length ${...}, received has length: ${issue.received.length}`
  );
```

### Bitfield

Validates Discord bitfield flags:

```typescript
import { bitfield } from '@discordkit/core';
import * as v from 'valibot';

enum MessageFlags {
  CROSSPOSTED = 1 << 0,
  IS_CROSSPOST = 1 << 1,
  SUPPRESS_EMBEDS = 1 << 2,
  SOURCE_MESSAGE_DELETED = 1 << 3
}

const messageFlag = bitfield('MessageFlags', MessageFlags);

const schema = v.object({
  flags: messageFlag
});

// Valid
v.parse(schema, {
  flags: MessageFlags.CROSSPOSTED | MessageFlags.SUPPRESS_EMBEDS
});
```

**Implementation:**

```typescript
export const bitfield = <TName extends string>(
  name: TName,
  flags: Flags,
  message: string = `Invalid Bitfield`
): SchemaWithPipe<...> => {
  const flagValues = Object.values(flags).filter(
    (flag) => !isNaN(Number(flag))
  );
  const mask = flagValues.reduce<bigint>(
    (total, flag) => total | BigInt(flag),
    0n
  );
  return pipe(
    custom<number | bigint | string, typeof message>(
      (val) =>
        isNonNullable(val) &&
        (typeof val === `number` ||
          typeof val === `bigint` ||
          isNumericString(val))
          ? (BigInt(val) & mask) === BigInt(val)
          : false,
      message
    ),
    title(name)
  );
};
```

## Error Handling

Validation errors provide detailed information:

```typescript
import { getGlobalApplicationCommandsSafe } from '@discordkit/client';
import { ValiError } from 'valibot';

try {
  const commands = await getGlobalApplicationCommandsSafe({
    application: 'invalid-id' // Not a valid snowflake
  });
} catch (error) {
  if (error instanceof ValiError) {
    console.error('Validation failed:');
    error.issues.forEach(issue => {
      console.error(`- ${issue.path}: ${issue.message}`);
    });
  }
}
```

### Using with tRPC

```typescript
import { summarize, ValiError } from 'valibot';
import { TRPCError } from '@trpc/server';

const loggedProcedure = baseProcedure.use(async (opts) => {
  const result = await opts.next();

  if (!result.ok) {
    // Log validation errors
    console.error(summarize(result.error.cause.issues));
  }

  return result;
});

export const router = t.router({
  getCurrentApplication: getCurrentApplicationProcedure(
    loggedProcedure,
    (cause) => {
      throw new TRPCError({
        code: `INTERNAL_SERVER_ERROR`,
        message:
          cause instanceof ValiError
            ? summarize(cause.issues)
            : `Failed to run procedure`,
        cause
      });
    }
  )
});
```

## Performance Considerations

Validation adds runtime overhead:

```typescript
// Fastest - no validation
const fast = await getGlobalApplicationCommands({ application: id });

// Slower - validates input and output
const safe = await getGlobalApplicationCommandsSafe({ application: id });

// Middle ground - validate input only
const custom = toValidated(
  getGlobalApplicationCommands,
  getGlobalApplicationCommandsSchema,
  null // Skip output validation
);
```

<Note>
  For most applications, the validation overhead is negligible compared to network latency. Use safe variants by default and optimize only if profiling shows validation is a bottleneck.
</Note>
