Get up and running with DiscordKit by making your first Discord API call.

## Prerequisites

- Installed DiscordKit and Valibot (see [Installation](/installation))
- A Discord Bot Token or User Bearer Token from the [Discord Developer Portal](https://discord.com/developers/applications)
- Node.js 20.x or higher

## Authentication setup

<Steps>
  <Step title="Configure your token">
    Set your Discord authentication token using the `discord.setToken()` method:

    ```typescript
    import { discord } from "@discordkit/client";

    // For Bot tokens
    discord.setToken(`Bot ${process.env.DISCORD_BOT_TOKEN}`, true);

    // For User Bearer tokens
    discord.setToken(`Bearer ${process.env.DISCORD_USER_TOKEN}`, true);
    ```

    <Note>
      The second parameter (`true`) marks the session as ready. Always validate your token before setting ready to `true`.
    </Note>
  </Step>

  <Step title="Store tokens securely">
    Never hardcode tokens in your source code. Use environment variables:

    ```bash .env
    DISCORD_BOT_TOKEN=your_bot_token_here
    DISCORD_USER_TOKEN=your_user_token_here
    ```
  </Step>
</Steps>

## Make your first API call

Here's a complete example that fetches guild information:

```typescript
import { discord, getGuild, getGuildSchema } from "@discordkit/client";
import * as v from "valibot";

// Configure authentication
discord.setToken(`Bot ${process.env.DISCORD_BOT_TOKEN}`, true);

async function fetchGuildInfo(guildId: string) {
  try {
    // Validate input
    const input = v.parse(getGuildSchema, {
      id: guildId,
      params: { withCounts: true }
    });

    // Make the API call
    const guild = await getGuild(input);

    console.log(`Guild: ${guild.name}`);
    console.log(`Members: ${guild.approximate_member_count}`);
    console.log(`Online: ${guild.approximate_presence_count}`);
    
    return guild;
  } catch (error) {
    if (error instanceof v.ValiError) {
      console.error("Invalid input:", error.message);
    } else {
      console.error("API error:", error);
    }
    throw error;
  }
}

// Usage
fetchGuildInfo("123456789012345678");
```

This example demonstrates:
- **Authentication** with `discord.setToken()`
- **Input validation** using Valibot schemas
- **Request execution** with the `getGuild` handler
- **Error handling** for validation and API errors

## Understanding the response

The `getGuild` function returns a `Guild` object with properties like:

<ResponseField name="id" type="string">
  Guild snowflake ID
</ResponseField>

<ResponseField name="name" type="string">
  Guild name (2-100 characters)
</ResponseField>

<ResponseField name="icon" type="string | null">
  Icon hash
</ResponseField>

<ResponseField name="approximate_member_count" type="number" required={false}>
  Approximate member count (only when `withCounts: true`)
</ResponseField>

<ResponseField name="approximate_presence_count" type="number" required={false}>
  Approximate online member count (only when `withCounts: true`)
</ResponseField>

## Rate limiting

DiscordKit automatically handles Discord's rate limits:

<Note>
  All requests are queued and processed according to Discord's rate limit buckets. You don't need to implement rate limiting yourself.
</Note>

```typescript
// These requests will be automatically queued and rate-limited
const guild1 = await getGuild({ id: "123456789012345678" });
const guild2 = await getGuild({ id: "987654321098765432" });
const guild3 = await getGuild({ id: "111222333444555666" });
```

## Common issues

<AccordionGroup>
  <Accordion title="401 Unauthorized error">
    - Verify your token is correct
    - Ensure you're using the right token type (`Bot` vs `Bearer`)
    - Check that your bot has been added to the guild
  </Accordion>

  <Accordion title="403 Forbidden error">
    - Your bot may lack the required permissions
    - Check the bot's role permissions in the Discord server
    - Some endpoints require specific OAuth2 scopes
  </Accordion>

  <Accordion title="ValiError: Invalid input">
    - Check that snowflake IDs are strings, not numbers
    - Validate required parameters are provided
    - Ensure parameter types match the schema
  </Accordion>
</AccordionGroup>
